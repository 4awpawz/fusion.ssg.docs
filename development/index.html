Project Development

The project is written in TypeScript and runs on Node.

The Name

fusion plays on the idea of seamlessly fusing together various technologies, e.g. HTML, Markdown, TypseScript Components, JSON Data, YAML Front Matter, etc., to seamlessly work together to simplify the creation of HTML documents and static websites.

Project Scope

The project's scope is limited to generating robust and resilient HTML documents and nothing else. It will not provide any other tooling other than what is absolutely necessary to accomplish that. Importantly, fusion.ssg requires no knowledge of front-end frameworks like React. This keeps the scope of the project focused, preventing feature creep, and benefits users who will be able to pick and chose their own tools.

Current Status

The project is still in its Alpha/prerelease stage. One or more Beta releases will follow and both Alpha and Beta releases should be considered unstable and transient. While in Beta, features and implementations might change dramatically and this should be considered the norm.

Versioning

fusion will use semantic versioning and will distinguish between development builds and release builds by their major version numbers

  1. Odd major version number are to be considered development versions.
  2. Even major version number are to be considered release versions.

What Has Already Been Implemented

  • Support for simple HTML document creation using Pages, Templates and Includes.
  • Support for simple token replacement involving front matter property values and matching property names in HTML.
  • Support for Typescript .tsx components, compilation and sand-boxed execution.
  • Support for document hydration using Typescript components and JSON data sources.
  • Support for generating collections of pages using Typescript components and JSON data sources.
  • Support for base URLs.
  • Support for development and release builds.
  • Support for content posting (e.g. Blogging)
    • Support for tags.
    • Support for categories.
  • Work in progress templates.
    • Individual files.
    • Folders.
    • Ignored.
  • End user project creation.
  • Configure website domain.

What Still Remains To Be Done

Longer Term Goals
