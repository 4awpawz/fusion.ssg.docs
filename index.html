Start by creating an HTML document using Pages, Templates, Includes, Front Matter and Tokens

Pages

Pages provide the overall layout of your HTML documents. They also act as containers for the content provided by Templates, Includes, Front Matter, Tokens and Typescript Components.

Templates And Front Matter

Templates can provide addtional content to Pages in two ways:

  • They can provide content specific to a document which replaces the document's template token, e.g. <body>><main>{template}<main><body>.
  • They can declare tokens, e.g. tokens: {pageTitle: All About Cats} in their front matter whose values replace the document's matching tokens, e.g. <h1>{pageTitle}<h1>.

See examples below.

Components

Use Typescript Components and JSON data sources for when your pages require more complex composition.

All the poswer of .tsx Components

When your HTML documents require more dynamic content you can use Typescript compoents along with JSON data sources that return markup.

Fusion compiles and runs components in its own sandboxed Typescript environment!

Create a component...

import { render } from "preact-render-to-string";

interface TeamMember {
    "name": {
        "first": string,
        "last": string
    }
}

interface Props {
    team: TeamMember[]
}

function formatHref(member: TeamMember): string {
    return `/team/${member.name.first}-${member.name.last}`;
}

function formatName(member: TeamMember): string {
    return `${member.name.first} ${member.name.last}`;
}

export const TeamList = function({ team }: Props): string {
    return render(
        <ul>
            {team.map(member => {
                return <li>
                    <span>
                        <a href={formatHref(member)}>{formatName(member)}</a>
                    </span>
                </li>;
            })}
        </ul>
    );
};

Add a component tag to your template or page or include.
