Why fusion.ssg?

Philosophy

fusion.ssg creates resilient HTML documents and static websites and gets out of your way when you are using it.

Minimal

fusion.ssg is built from the ground up to be just that and very quick too.

Familiar

fusion.ssg has a very low "barrier to entry" because it seamlessly employs concepts that you are already familiar with, like templates, pages, includes, markdown, tokens, and Typescript components.

Productive

No complicated tooling and configuration. fusion.ssg's project generator gets you up and running quickly.

Features

  • Simple HTML document creation using pages, templates, includes, Typescript components and JSON data sources.
  • Simple token replacement involving front matter property values and matching property names in HTML.
  • TypeScript component compilation and sand-boxed execution.
  • Document hydration using Typescript components and JSON data sources.
  • Multiple HTML document generation (i.e. Collections) using Typescript components and JSON data sources.
  • Development and release builds.
  • Base URLs for release builds.
  • Work in progress templates are ignored during release builds for individual files and folders.
  • Content posting (e.g. Blogging), including categories and tags.
  • Beautified HTML documents.
  • Project generator rapidly creates new projects requiring no initial configuration to begin building your site.

Create An HTML Document From A Simple Component That Generates A List Of Team Members

The team.json data source.

[
{"name": { "first": "John", "last": "Doe" }},
{"name": { "first": "Jane", "last": "Doe" }},
{"name": { "first": "Julia", "last": "Doe" }}
]

A template with a simple TeamMembers component tag that declares team.json as a data source.

---
tokens: {
    pageTitle: Team Members
}
---
<h1>team</h1>
<p>Thanks to our team our rollout has been a huge success.</p>
<h2>contributors</h2>
<TeamMembers datasources="team.json" />

The simple TeamMembers component implementation.

interface TeamMember {
    "name": {
        "first": string,
        "last": string
    }
}

interface Props {
    team: TeamMember[]
}

function formatName(member: TeamMember): string {
    return `${member.name.first} ${member.name.last}`;
}

export const TeamMembers = function({ team }: Props): string {
    return (
        <ul>
            {team.map(member => {
                return <li>
                    <p><{formatName(member)}</p>
                </li>;
            })}
        </ul>
    );
};

The Name

fusion plays on the idea of seamlessly fusing together various technologies, e.g. HTML, Markdown, TypseScript Components, JSON Data, YAML Front Matter, etc., to seamlessly work together to simplify the creation of HTML documents and static websites.

Project Development

The project is written in TypeScript (no excuses, it just works beautifully) and runs on Node.

Project Scope

The project's scope is limited to generating robust and resilient HTML documents and nothing else. It will not provide any other tooling other than what is absolutely necessary to accomplish that. Importantly, fusion.ssg requires no knowledge of front-end frameworks like React. This keeps the scope of the project focused, preventing feature creep, and benefits users who will be able to pick and chose their own tools.

Current Status

The project is still in its Alpha/prerelease stage. One or more Beta releases will follow and both Alpha and Beta releases should be considered unstable and transient. While in Beta, features and implementations might change dramatically and this should be considered the norm.

Versioning

fusion will use semantic versioning and will distinguish between development builds and release builds by their major version numbers

  1. Odd major version number are to be considered development versions.
  2. Even major version number are to be considered release versions.

What Has Already Been Implemented

  • Support for simple HTML document creation using Pages, Templates and Includes.
  • Support for simple token replacement involving front matter property values and matching property names in HTML.
  • Support for Typescript .tsx components, compilation and sand-boxed execution.
  • Support for document hydration using Typescript components and JSON data sources.
  • Support for generating collections of pages using Typescript components and JSON data sources.
  • Support for base URLs.
  • Support for development and release builds.
  • Support for content posting (e.g. Blogging)
    • Support for tags.
    • Support for categories.
  • Work in progress templates.
    • Individual files.
    • Folders.
    • Ignored.
  • End user project creation.
  • Configure website domain.
