GETTING STARTED

Synopsis

Imagining fusion.ssg As An Assembly Line

If you are new to fusion.ssg it will be advantageous for you to begin learning how to use it by having a conceptual understanding of the process that transforms your project's files into HTML documents. This will make learning fusion.ssg even easier. Therefore, let us begin your journey by imagining an assembly line that is responsible for creating a single HTML document. It has a beginning and an end. Along that assembly line there are numerous points where operations are performed on what is traveling on it. The following describes the operations at each point on that assembly line:

Gather All Templates

Accumulates all the necessary items that will be needed to start producing the finished products. These items are the files of your project, but not all the files, only the ones called templates, which are located in a project's src/templates folder. You can think of templates as the DNA of a potential HTML document. In its DNA it carries almost all the information needed to build one HTML document. This process passes each template that it has found down the assembly line, one at a time, for further processing.

Create A Developing HTML Document

Analyzes the content of the template file, insuring that it contains the required YAML front matter and possibly also content in the form of HTML or markdown. In the template's front matter it looks for a property called page whose value points to a file located in a project's src/pages folder which this template file is associating itself with. The page file is composed entirely of HTML and it exists to provide the overall structure of your finished HTML document. If fusion.ssg doesn't find a page property, it defaults to using the src/pages/default.html page. If the template has content, fusion.ssg searches the page's content for a "{{template token}}", and if it finds it, replaces it with the template's content. At this point, we have a developing HTML document.

Apply Component To The Developing HTML Document

This is where things get interesting. fusion.ssg searches for all the component tags, e.g. <MyComponent />, in the developing HTML document's content, which now contains the content contributed by the page and possibly by the template. If it finds one or more component tags, fusion.ssg analyzes their properties to discover if they declare their dependency on one or more JSON data sources residing in the project's src/data folder. If they do, these data sources are now acquired and each component is then called, being passed their data sources along with any other properties that the component tag may have. As each component returns its content in the form of HTML, fusion.ssg replaces the component's respective tag in the developing HTML document with that content.

Apply Template Tokens To The Developing HTML Document

fusion.ssg now searches the template's front matter for any tokens that it may have declared and, if found, replaces any "{simple token}" s found in the developing HTML document's content with their respective values.

Serialize The HTML Document

At this point the HTML document that has been under development is 100% complete. fusion.ssg now serializes it to its respective file in the project's build folder.

While the actual process that fusion.ssg uses is quite a bit more complex than what this very high-level overview provides, you should now have a very good conceptual understanding of how fusion.ssg goes about its business of producing HTML documents.

The remaining documentation relies heavily on examples. It assumes that you already have some working knowledge of HTML, markdown, YAML front matter and TypeScript components.
