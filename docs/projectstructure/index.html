GETTING STARTED

Project Structure

fusion.ssg projects will contain the files and folders that are depicted below.

fusion.ssg's project generator relieves you of the burden of having to manually create and configure your projects.


  my-app/
  ├─ .git/
  ├─ .meta/
  ├─ build/
  ├─ lib/
  ├─ node_modules/
  ├─ src/
  │  ├─ components/
  │  ├─ data/
  │  ├─ etc/
  │  ├─ includes/
  │  ├─ media/
  │  ├─ pages/
  │  ├─ scripts/
  │  ├─ templates/
  │     ├─ posts/
  ├─ .eslintignore
  ├─ .eslintrc.json
  ├─ .gitignore
  ├─ .assets.json
  ├─ fusion.json
  ├─ package-lock.json
  ├─ package.json
  ├─ tsconfig.json

Project Structure Details

Folder Description
.git Used by Git.
.meta Contains metadata used internally by fusion.ssg during each build cycle.
build Contains the output of each build cycle.
lib Contains compiled components.
node_modules Contains your project's Node packages.
src Contains the folders and files used to build your project.
src/components Contains components.
src/data Contains JSON data sources used by components.
src/etc Contains files that will be moved to the root of the build folder at the end of each build cycle, example 404.html.
src/include Contains your project's include files.
src/media Contains your project's media files.
src/pages Contains your project's page files.
src/scripts Contains your project's browser script files.
src/templates Contains your project's template files.
src/templates/posts Contains post markdown files.
.eslintignore Contains folders and files to be ignored by eslint.
.eslintrc.json eslint configuration file.
.gitignore Contains folders and files to be ignored by Git.
.assets.json Contains the meta data generated after each build cycle.
fusion.json Project's configuration file.
package-lock.json NPM lock manifest.
package.json NPM package manifest.
tsconfig.json Typescript configuration file.
