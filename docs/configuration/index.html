Configuration

The following configuration options are available for you to override in your project's src/fusion.json file.

postsFolder: The name to use for the folder that hosts your site's posts.

  • default: "posts"

Example:

"postsFolder": "blog""

baseURL: If your site is served from a subfloder, such as is the case for sites hosted on github.io, prepend {baseURL} to your site's root relative URLs and they will be replaced with this value during release builds.

  • default: ""

Example:

Set baseURL


"baseURL": "fusion.ssg.docs"

and prepend baseURL to your site's root relative URLs


<a href="{baseURL}/documentation">Docs</a>

and during release builds, {baseURL} will be replaced by its value.


<a href="fusion.ssg.docs/documentation">Docs</a>

wips: These are files and folders of templates that are to be ignored during release builds.

  • default: []
"wips": ["src/templates/docs/next-version", "!src/templates/docs/next-version/index.md"]

Files and folders prepended with "!" are negated.
