HTML DOCUMENTS

Introduction

Visualizing how fusion.ssg creates HTML documents

As a developer it is important to be able to visualize the processes that fusion.ssg performs to transform your project's files into HTML documents. As easy as fusion.ssg is to understand, this will make using fusion.ssg even easier. Therefore, imagine an assembly line that is responsible for creating a single HTML document. It has a beginning and an end. Along that assembly line there are numerous processes that operate on what is traveling on that assembly line.

  1. The first process on that assembly line accumulates all the necessary items that will be needed to start producing the finished products. In fusion.ssg's case, those items are the files of your project, but not all the files, only special ones called templates. You can think of templates as the DNA of a potential HTML document. In its DNA it carries almost all the information needed to build one HTML document. This process passes each template that it has found down the assembly line one at a time for further processing.

  2. The second process on that assembly line analyzes the content of that template file, insuring that it contains the required YAML front matter and possibly also content in the form of HTML or markdown. In the template's front matter it expects to find a property called page, whose value is a path which points to another special kind of file called, you guessed it, a page. That page is composed entirely of HTML and it exists to provide the overall structure of your finished HTML document. Now that a page has been associated with the template, they both travel down the assembly line together to the next process.

  3. The third process on that assembly line searches the page for something that it can recognize as a template token, which indicates where in the page to place any content that the template may contain. If it finds that token, it will replace it with any content it finds in the template. The third process is now complete and both the template and the page continue to travel down the assembly line together to the next process.

  4. The fourth process on that assembly line is where things begin to get interesting, and it searches for any and all custom component tags in the page which, remember, now contains any content that the template may have contributed to it. If it finds any custom components, it analyzes their properties to discover if they declare their dependency on one or more JSON data sources. If they do declare their dependency on JSON data sources, these too are now acquired and each custom component is now called in turn, being passed any data source they may have declared as a dependency. As each custom component returns the markup that it created, the process replaces that component's tag in the page with that markup.

At this point you may be wondering why the template is still proceeding down the assembly line along with the page and asking yourself, "What additional value can it provide to the future HTML document that will be created from the page's content?" The next process on the assembly line will answer that!

  1. The fifth process on the assembly line searches the template's front matter for any and all front matter tokens that it may declare and if found will use those tokens' values to replace their matching tokens in the page's content, which now includes not only the content contributed by the template, if any, as well as the content contributed by all the custom components as well.

  2. The sixth and last process on that assembly line writes the page's content out to an HTML file, then "recycles" the page so that it can be used again and finally "trashes" the template as templates can only be used to create one HTML document.

While the actual processes that fusion.ssg uses are quite a bit more complex than the previous high-level overview, you should now have a fairly good understanding of how things fit into place, so to speak. The remaining pages in this section of the documentation will go into greater detail, not only covering the items introduced here like pages, templates, etc., but all the other items that are available to you to use to create HTML documents.
