HTML DOCUMENTS

Components

Required: No
Type: Typescript, i.e. .tsx
Project Location: src/components

Components are compiled and called asynchronously and run in a sand-boxed environment.

Properties

Components can reference their properties as {[property name]} (see code example below).

There are two predefined component properties recognized by fusion.ssg:
  • isCollection: informs fusion.ssg that this component should be called to generate a collection of HTML documents rather than rendering output for a single HTML document.
  • dataSources: is a comma separated lists of JSON file names that are located in your project's src/data folder. At runtime, fusion.ssg reads each data source, converting them to JavaScript objects and passes the converted data sources to your component along with two additional properties (see below) that may be useful. Data sources passed to your components can be destructured from props using their JSON file naame minus the .json extension.
In addition to your component's declared properties, fusion.ssg also passes two additional properties to your components:
  • asset: the metadata that fusion.ssg has generated for the HTML document that this component is being called for.
  • assets: a list containing the metadata that fusion.ssg has generated for all HTML documents in your project.

fusion.ssg serializes the metadata that it has generated for your project to a file named assets.json in your project's root folder.

Example: A component that receives a data source that is returned as a list. 
interface TeamMember {
    "name": {
        "first": string,
        "last": string
    }
}

interface Props {
    team: TeamMember[]
}

function formatHref(member: TeamMember): string {
    return `/team/${member.name.first}-${member.name.last}`;
}

function formatName(member: TeamMember): string {
    return `${member.name.first} ${member.name.last}`;
}

export const TeamList = function({ team }: Props): string {
    return (
        <ul>
            {team.map(member => {
                return <li>
                    <span>
                        <a href={formatHref(member)}>{formatName(member)}</a>
                    </span>
                </li>;
            })}
        </ul>
    );
};
Example: A template that contains a component tag with a dataSources property. 
<h1>Team</h1>

<p>Thanks to our team, our rollout has been a success.</p>

<h2>Contributors</h2>
<TeamList dataSources="team.json" />
