HTML DOCUMENTS

Collections

Required: No
Type: Typescript, i.e. .tsx
Return Type: string
Project Location: src/components

Why would you want to create a collection of pages? Imagine you have a portfolio of one kind or another and you want to create pages for all the items contained in your portfolio. Imagine having to do that manually for each page. That would be quite tedious and laborious. fusion.ssg provides a much better solution.

Collection templates and components work together to provide an easy to understand and a simple to implement mechanism for creating collections of pages. Simply put, fusion.ssg calls collection components that return content, just like normal components do, but fusion.ssg calls these components repeatedly until they return no content. In addition, for each call, the component receives an index along with any of the other properties the component has declared as well as the default properties they normally receive, such as asset and assets. The value of this index is initially zero and is incremented for each subsequent invocation of the component. The index, for example, can be used to determine which item in a data source is to be used to produce the current HTML document and when the index value is greater than the number of items in the data source the component returns nothing, which signals to fusion.ssg that there are no more documents to be created.

Collection Templates

A collection template is any template that declares a front matter property named isCollection with a value of true.

fusion.ssg automatically adds this property with a value of false to all non collection templates.

Example: A collection template. 

---
page: team-member
⋮
isCollection: true
---
<h1>team</h1>

<p>thanks to our team, our rollout has been a success.</p>

<h2>contributors</h2>
⋮

A collection component is any component that declares a property named isCollection.

Example: A collection template with a collection component tag. 

---
page: team-member
⋮
isCollection: true
---
<h1>team</h1>

<p>thanks to our team, our rollout has been a success.</p>

<h2>contributors</h2>
<TeamMember isCollection datasources="team.json" />
⋮

Collection component tags are applicable within includes, templates and pages.

Collection Components

Arguments

Besides the properties declared by the component, it will, like every component, also receive asset and assets.

  • index: number, initially set to 0 and incremented by 1 for every subsequent invocation.

Returns

When collection components have content to contribute to a page, they return an object with properties for the content, the title, and the name of the document:

  • content: string, the content to contribute to this HTML page.
  • title: string, will replace the simple title token in the HTML document's title tag, e.g. <title>{title}</title>.
  • htmlDocumentName: string, the name for this HTML page.

When collection components no longer have content to contribute to a page, they return nothing.

Example: A collection component implementation. 
interface TeamMember {
    name: {
        first: string,
        last: string
    },
    address: {
        street: string,
        apt: string,
        city: string,
        state: string,
        zip: number,
    }
}

interface Props {
    team: TeamMember[]
}

export const TeamMember = function(index: number, { team }: Props): { content: unknown, title: string, htmlDocumentName: string } | undefined {
    if (index === team.length) return;
    const content = (
        <>
            <h2>{team[index].name.first} {team[index].name.last}</h2>
            <ul style="list-style: none">
                <li><span style="color: green">{team[index].address.city}, {team[index].address.state}</span></li>
            </ul>
        </>
    );
    const title = `${team[index].name.first} ${team[index].name.last}`;
    const htmlDocumentName = `${team[index].name.first}-${team[index].name.last}`;
    return { content, title, htmlDocumentName };
};
